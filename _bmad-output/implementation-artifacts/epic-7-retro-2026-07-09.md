## 2. What went well

- **The standing "real-provider LLM smoke" DoD finally triggered — and earned its keep.** Open since
  Epic 5 AI#2, no trigger in Epic 6. Story 7.2 introduced the first new LLM path (enrichment); the
  env-gated smoke (`ENRICHMENT_SMOKE=1`) **caught a real config bug** — `provider: 'anthropic'` with a
  non-Anthropic-shaped `ENRICHMENT_LLM_API_KEY` — that fake-green unit tests (injected fakes, never
  mocking `BaseChatModel`) would have sailed past. **Borja named this a top win.**
- **The Epic-6 operational backlog (AI#1) worked exactly as designed.** Every deferral in 7.2/7.3
  landed with an explicit home (P2.2 DLQ, P2.3 messageDeleteBulk, P2.5 lifecycle notification), and
  debt was *promoted to real stories*: P1.1 → `ops-1-maxlen-stream-trimming.md`, P1.2 →
  `ops-2-integration-test-isolation.md` (done). Deferred work stopped dying silently. **Borja named
  this a top win.**
- **"Plumbing first, semantics after" kept risk low.** 7.1 moved contracts/model/config; later stories
  were semantics only. Boring reuse → a low-drama epic despite a destructive rewrite. **Borja named
  this a top win.**
- **Adversarial review caught real defects, not just noise.** The 7.2 P1 HIGH (a `throw` escaping
  `fetchUrl` and poisoning the message, on the SSRF path) was genuine and pre-merge.
- **Migration executed cleanly.** Destructive TRUNCATE runbook + strict link contracts
  (`isHttpUrl`, no `z.string().url()`) landed 0 incidents. *(Borja notably credited the systems over
  the outcome — see §5.)*

## 3. Challenges

- **🟡 Blind Hunter severity noise (Borja's concern).** The Blind Hunter review layer produced
  inflated-severity false positives in essentially every story: 7.1 *"backend won't boot"* (false —
  all services use `env_file: - .env`), 7.3 a *"High"* empty-purge (dismissed — ratified + bot fetches
  full content), 7.6 *diff-only* FPs (neutralized by the Edge Case Hunter). Value concentrated in the
  Edge Case Hunter and the Acceptance Auditor. A Blind-only "High" enters triage at the same weight as
  an Auditor finding and then collapses — costing a second pair of eyes every time.
- **🟡 Residual load-sensitive integration intermittency (Borja's concern).** The *original* RBAC
  "test-guild role leak" was **fixed** in `ops-2` — the spike proved it a **stale assertion** (from
  PR #32's @everyone/guild-id injection), not a flake; run-unique ids + fail-fast guard +
  `development_guide` precondition closed it. But 7.4 (2026-07-09) still documents a *separate*
  load-sensitive intermittency (auth/documents/search under heavy load) as "pre-existing — not yours."
  The leak is closed; the load-sensitive class is not. The **habit** of writing "not yours" was
  correct here, but it is the habit worth watching.
- **🟡 "Final" turned out conditional.** Epic 6 declared the roadmap done; a product pivot reopened it
  a week later. Not a failure — but "complete" is a product decision, not a permanent state.

*(Note: MAXLEN, a four-retro worry going in, is **not** a challenge — it was DONE via `ops-1`
(PEL-safe `XTRIM … MINID`, merged PR #39, 2026-07-08) before Epic 7's stories. 7.2/7.3 correctly left
the bare `xAdd` producers untouched; the trimmer is a dedicated centralized loop. See §4.)*

## 11. Key takeaways

1. **The standing DoD's first live trigger validated six retros of process debt.** The real-provider
   smoke caught a config bug on its very first LLM path — fake-green would have missed it.
2. **Homed debt gets paid; homeless debt dies.** Epic 6's backlog turned both its P1 items into real,
   *merged* ops stories — MAXLEN (`ops-1`, PR #39) and integration isolation (`ops-2`, PR #41) — before
   Epic 7's stories even started. The backlog didn't just record debt; it discharged it.
3. **Credit the system, not the heroics.** A destructive migration with 0 incidents came from the
   plumbing-first reuse, the smoke DoD, and the backlog — repeatable on purpose, not luck.
4. **"Fixed" and "documented" are different.** The RBAC leak was *fixed* (stale assertion); the
   load-sensitive flake is only *documented*. "Not yours" must cite a tracked home.
5. **"Complete" is conditional.** A declared-closed roadmap reopened via a product pivot. Plan for the
   possibility that "final" is a product decision, not a terminal state.
